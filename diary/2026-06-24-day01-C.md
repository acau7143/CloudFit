## 왜 이 기술/방법을 선택했는가
| 항목 | 선택한 것 | 대안 | 선택 이유 |
|------|-----------|------|-----------|
| 인증 방식 | ADC (gcloud auth application-default login) | JSON 서비스 계정 키 | 조직 정책으로 JSON 키 생성 차단됨. ADC가 더 안전하고 실무 권장 방식 |
| 인스턴스 사양 | e2-micro | e2-small, n1-standard | 메트릭 수집 테스트 목적이라 최소 사양으로 충분. 비용 최소화 |
| 인스턴스 생성 방식 | GCP 콘솔 GUI | gcloud CLI | 초기 설정은 GUI가 직관적이고 실수가 적음 |

## 오늘 처음 안 사실
| 몰랐던 것 | 알게 된 것 | 왜 그렇게 되는지 |
|-----------|-----------|----------------|
| JSON 키가 왜 막혔는지 | GCP가 2023년부터 신규 계정에 `iam.disableServiceAccountKeyCreation` 정책을 자동 적용 | JSON 키 유출로 인한 보안 사고가 많아서 GCP가 기본값으로 차단함 |
| ADC가 뭔지 | gcloud로 로그인해두면 Python 코드가 자동으로 인증 정보를 가져다 쓰는 방식 | 별도 키 파일 없이 로컬 인증 정보를 참조하기 때문 |
| 무료 계정 → 일반 계정 차이 | 크레딧 소진 후 자동 과금됨 | 무료 계정은 크레딧 소진 시 서비스 중단, 일반은 카드 자동 청구 |
